Instant Apps

Frost Finder Experience Builder

Pros:

  • Very easy to take a map from AGOL and create the app.
  • Has a search feature for specific fields within specific layers.
  • Quick to add new layers for different events, just through updating the web map being used.

Cons:

  • Lacks any form of customization in the use of the map. Really just displays an already-made map with very surface-level interactions. Including only default pop-ups.
  • Slow to load if using multiple layers.
  • No way to customize the display of the map or app. You can really only change the colours in the header and add a logo.
  • The only way to make custom layers, and edit them is to add them to the made web map and re assing the map within instant apps.
  • Not very user friendly for toggling between different layers.

Conclusion

While Instant Apps is very useful for quick production and display of a map, it isn't the right tool for this job. The interface lacks the customization we are looking for to achieve something that is both simple and straightforward to use. The current default interface lacks design and gives the user too many options. Similarly, the lack of options to customize the search function, in both zoom extent and the look of the pop-up. The pop-ups once again give the user too much data, showing all fields within the layer. The layers option tab is also a user interface issue, once again giving too many options for the user. For the most part, the aspects of Instant Apps which may prove useful for the project can be found in Experience Builder, such as the search bar and updating the app by updating the web map being used.
